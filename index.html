YouTubeTranscriptGen.com LIVE

Production web app providing full, multi‑language YouTube transcripts with a streaming AI assistant for question answering & summarization. Emphasis on resilience and UX speed.

  • Transcript extraction + language switching (cached, fallback tolerant)
  • Streaming AI chat (incremental markdown, scroll anchoring)
  • Exports: TXT / SRT / VTT / JSON; ratings + credit economy
  • Proxy rotation & warm path logic to mitigate rate / geo constraints
  • Stripe billing + structured data SEO + persisted conversations

Tech focus: Efficient transcript retrieval pipeline, resilient network layer with rotating proxies, incremental SSE rendering for near real‑time UX, token cost control strategies.

Challenges solved: Handling regional blocking, balancing model latency vs quality, preventing scroll jank during long streaming outputs.

Impact: Enables users to extract and query long-form video knowledge in seconds instead of manually scrubbing hours of content.

Retrieval‑Augmented QA Prototype

Experimental pipeline for answering domain questions over large text corpora (e.g. legal). Focus on retrieval relevance & latency budget control.

  • Document chunking + embedding & vector store indexing
  • Hybrid similarity retrieval + context window packing heuristics
  • Deterministic assembly layer feeding generation model
  • Instrumentation for recall / precision & token efficiency

Approach: Hybrid scoring (semantic + term features), rank truncation, adaptive window packing based on token budget.

Evaluation: Early harness for manual relevance judgments + recall tracking across query set.

Next: Add multi‑vector representation and shallow re‑rank pass.

Router Data Plane Simulator (C)

Low‑level packet processing path built over Mininet to internalize protocol behavior & failure modes.

  • Ethernet frame parsing, ARP cache + pending queue
  • IPv4 TTL decrement & checksum recomputation
  • Longest Prefix Match routing decisions
  • ICMP: unreachable, time exceeded, echo reply logic

Focus: Reinforce deterministic packet path reasoning, timing of ARP resolution vs queued packets, and correctness of checksum updates.

Learning: Deepened understanding of failure surfaces (ARP exhaustion, TTL expirations) & diagnostics flows (ICMP signalling).

Outcome: Better intuition for latency sources and error propagation across network layers.
